# Accessibility QA

## Scope

Use this checklist for CreditCrest AI web screens and CrestLend mobile/web preview screens.

## Visual Accessibility

- [ ] Text has sufficient contrast against backgrounds
- [ ] Body copy remains readable on mobile
- [ ] Buttons and badges do not truncate important labels
- [ ] Focus, selected, error, and disabled states are visually distinct
- [ ] Color is not the only way to understand risk or status

## Interaction Accessibility

- [ ] Primary actions are reachable by keyboard where applicable
- [ ] Links and buttons have clear accessible names
- [ ] Form fields have clear labels
- [ ] Error messages explain how to recover
- [ ] Touch targets are large enough on mobile

## Content Accessibility

- [ ] Disclaimers are understandable without legal expertise
- [ ] Educational copy avoids official score claims
- [ ] Risk labels use directional language
- [ ] Empty states explain what the user can do next

## Assistive Technology Smoke Check

- [ ] Screen reader order follows the visual flow
- [ ] Navigation labels are meaningful
- [ ] Icon-only controls have accessible labels or adjacent text
